## Context

The game targets 3 languages at launch: English (primary), Japanese, Portuguese.
News feed templates (~375 × 3 = ~1,125 strings), UI strings (~500 per language),
and idol names (kanji + hiragana + romaji) must all be translated. Language switch
must apply without restarting the game.

The project already uses `svelte-i18n` with locale files in `src/lib/i18n/`.

## Decision

### svelte-i18n with Lazy-Loaded Locale Files

Keep the existing `svelte-i18n` setup and formalize conventions:

```
src/lib/i18n/
├── en.json          # English (primary, complete)
├── ja.json          # Japanese
├── pt.json          # Portuguese
└── index.ts         # Init + lazy loader
```

### 1. String Organization

```json
{
  "nav": { "portal": "Portal", "roster": "Roster", ... },
  "portal": { "title": "Dashboard", "news": "Latest News", ... },
  "roster": { "columns": { "name": "Name", "tier": "Tier", ... } },
  "common": { "confirm": "Confirm", "cancel": "Cancel", ... },
  "news": { "templates": { "T001": "{idol} shines on {show}!", ... } }
}
```

**Rules:**
- Keys are dot-separated: `$t('roster.columns.name')`
- Screen-specific strings under screen key: `portal.*`, `roster.*`
- Shared strings under `common.*`
- News templates under `news.templates.*` (keyed by template ID)
- No hardcoded user-facing strings in components — all via `$t()`

### 2. Idol Names

Idols have 3 name representations stored in IdolCore:

```typescript
interface IdolCore {
  nameRomaji: string;    // "Tanaka Yui" — used in EN
  nameJp: string;        // "田中ゆい" — used in JA
  // PT uses nameRomaji (same as EN)
}
```

Components display name based on current locale:
```svelte
{$locale === 'ja' ? idol.nameJp : idol.nameRomaji}
```

### 3. News Template System

News templates are identified by a stable template ID (e.g., `T001`). Each
language file contains all templates. The news generator fills placeholders
at event-fire time:

```typescript
// Template: "{idol} receives standing ovation at {venue}"
function generateNews(templateId: string, vars: Record<string, string>): string {
  const template = $t(`news.templates.${templateId}`);
  return template.replace(/\{(\w+)\}/g, (_, key) => vars[key] ?? key);
}
```

### 4. Language Switch

Language change applies immediately without restart:
1. User selects language in settings
2. `svelte-i18n` locale is updated
3. All `$t()` calls reactively update (Svelte subscription)
4. Preference saved to localStorage (persists across sessions)

### 5. Number and Date Formatting

- Numbers: `Intl.NumberFormat` with locale (¥1,000,000 in EN, ¥1.000.000 in PT)
- Dates: Week/month display uses localized short names
- Percentages: locale-aware decimal separator

### 6. RTL Support

Not needed for launch languages (EN, JA, PT are all LTR). Architecture does
not preclude RTL addition later — Tailwind supports `dir="rtl"` via plugin.
